# Course Close — Exit Ticket

Companion to [`lesson.md`](lesson.md) Part 11. Structure unchanged from the previous wrap-up (see [`../../../design/decision-log.md`](../../../design/decision-log.md)) — still required per `../../../CLAUDE.md` "Assessment and participant outputs."

Each participant submits, in writing, before the session closes:

1. **One calibration insight** you will use going forward (e.g., a specific diagnostic you now check before trusting a baseline, or a lesson from today's sensitivity-analysis exercise).
2. **One scenario-design practice** you will keep (e.g., always comparing within the same emissions pathway; always writing the metadata block before running a scenario).
3. **One reserve-requirement or policy-analysis habit** you will apply (e.g., explicitly labeling illustrative-proxy numbers; always stating a caveat).
4. **One remaining question** for future support.

## Collection and use

Facilitator compiles all four responses per participant into a shortlist of:
- common insights/practices worth reinforcing in any future refresher, and
- common remaining questions to route into [`../../../qa/sme-questions.md`](../../../qa/sme-questions.md) if institutional.

Unlike the previous wrap-up structure, there is no separate live "remaining questions" discussion block today — questions raised here go directly into the asynchronous tracking in `qa/sme-questions.md`.
